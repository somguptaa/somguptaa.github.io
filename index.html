Project 01 · Full Stack · Featured

Food Order Management System

Full-stack application with a Spring Boot REST API backend and a dark-theme single-page frontend served via Spring Boot static resources — no external JS framework. Implements a strict 5-stage order lifecycle with domain-level enforcement and comprehensive validation.

  • 6+ RESTful APIs covering full CRUD and a PATCH status endpoint enforcing a 5-stage order lifecycle: PENDING → CONFIRMED → PREPARING → DELIVERED / CANCELLED
  • Enum-level canTransitionTo() guards rejecting illegal state changes with structured 409 Conflict responses — enforcing real-world business rules at the domain layer
  • Bean Validation (@Valid, @NotBlank, @Min, @Positive) with a RestControllerAdvice Global Exception Handler returning field-level errors on every 400 response
  • All REST endpoints documented using SpringDoc OpenAPI with Swagger UI for interactive testing and clear API contract visibility
